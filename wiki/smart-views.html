Smart Views & Home

Your notes are written once but readable from every angle: by person, by project, by date, by task, by file. Nothing here needs maintaining — it’s all derived from what you wrote.

The lens switcher

Open Smart View from the sidebar. A floating bar of round lens buttons sits over the content — hover one to see its label, click to switch. One lens is active at a time.

LensShows
HashtagsEvery # tag in the vault, sorted by open work.
MentionsEvery @ mention, same treatment.
Person / Team / Company / Location / ProductOnly mentions of that type.
Project / Idea / Topic / StatusOnly hashtags of that type.
TasksEvery checkbox in the vault, open items first, grouped by notebook.
DatesA day-by-day agenda of everything dated (details below).
FilesEvery imported file, grouped by notebook (details below).
AlertsThe notifications view — overdue and upcoming reminders and tasks.
EntitiesA browsable directory of everything you’ve tagged — tabs per type, avatars and note counts, one click to the full entity card.
StatsA build-your-own dashboard of your vault: bento-style widgets, ten one-click presets, and a Quick or Advanced builder for custom counts and charts.

Tag cards

Inside a lens, each tag gets a card:

  • The colored chip with the tag’s name, type and note count.
  • An open-work badge (“3 open”) when there are unfinished items.
  • A progress bar tracking done vs. total.
  • Every block that mentions the tag, as a checkable row — checkbox, snippet, and source notebook. Click the text to jump to the note; tick the box to complete the block right there (see Complete & incomplete).
  • Completed rows tuck under an “N completed” toggle so the card stays focused on what’s open.

The Dates lens

A single agenda merging dated tasks (checkboxes with due dates) and dates mentioned in prose (“meet Sara friday”) — deduplicated so nothing appears twice. Items group into sticky buckets: Overdue (red), Today, Tomorrow, This week, This month, Later. Every row has a checkbox and a link back to its source note.

The Files lens

Everything you’ve attached, grouped by notebook:

  • Click a file to open it in its default app.
  • Each row shows name, size, type and an indexing state: Embedded (text extracted and searchable), Processing, or No text (stored, nothing to extract). Search over file contents is instant, full-text and fully offline.
  • Delete (hover for the trash icon) removes the file, its extracted text and knowledge-graph entries, and the file pill in the note — one action purges everything, with a confirmation first.

Supported file types and how extraction works

Text is extracted immediately at import time — not deferred — on a background thread so the app stays responsive. The extracted text is stored in the vault alongside the original file and feeds the knowledge graph without any extra step.

File types that produce searchable, indexed text:

FormatExtensions
Plain text & markup.txt .md .markdown .text .log .json .csv .tsv
Web.html .htm (script/style blocks stripped)
PDF.pdf
Microsoft Word.docx
Microsoft PowerPoint.pptx (slide text + speaker notes, in slide order)
Spreadsheets.xlsx .xls .xlsb .ods (all sheets, real cell values)
Rich Text.rtf
OpenDocument.odt .odp

Files of any other type (images, video, audio, archives, etc.) are stored and attached as-is — the pill appears in the note and the file opens in its default app, but no text is extracted and no knowledge-graph entities are derived from the content.

A scan-only PDF (no embedded text layer) stores cleanly but shows No text in the Files lens — the same as an unsupported type. The original file is always kept intact regardless of extraction outcome. The import size limit is 10 MB per file.

Home — your daily agenda

Home greets you (in English plus a rotating second language with a phonetic guide), shows today’s date, and lays out your day — computed live from your notes, no AI required and nothing to refresh:

  • Active reminders — what’s counting down right now.
  • Overdue — red rows with “done? · snooze?” shortcuts.
  • Today, then the next few days grouped per day, then Later.
  • Recently completed — shown when you’re all caught up, so an empty day feels earned.

Use the small tabs at the top right to regroup the agenda by Time (default), @Mentions or #Tags — the same items organized around people and projects instead of the clock.

The note cover graph

Each notebook opens with a small interactive knowledge-graph panel above the editor — a scoped view showing the entities found in that note, their typed relations, and up to five semantically related notes orbiting the central hub.

  • Drag-pan — click and drag anywhere on the graph to reposition it.
  • Node drag — drag an individual node to pin it somewhere on the canvas; the force layout adapts around it.
  • ⌘+scroll (or Ctrl+scroll) to zoom in and out — plain scroll is reserved for the note itself, so the modifier key keeps both actions unambiguous.
  • Click a node to open its entity profile panel (showing attributes, related notes, and all blocks that mention it). Clicking a related-note node opens that notebook. Clicking the central hub node does nothing.
  • Hover a node for a small tooltip with the entity's name, type, and sentiment indicator.
  • Cluster-by legend — the legend strip in the top-right corner lists the entity types present in the note. Clicking a type recolors the graph by that type, grouping nodes into clusters. Click again or pick a different type to switch; no selection removes clustering.

The cover graph scrolls away naturally as you move into the note — it is part of the scroll flow, not a fixed overlay. Once it has scrolled out of view the force simulation pauses automatically to save resources; scrolling back up remounts and relays out the graph. The compact entity-kind strip that appears in the app titlebar once the cover is hidden reflects the same entity types.

There is no button to hide the cover graph permanently — scrolling past it is the intended pattern. The graph only appears when there is at least one entity in the note.

The Smart Scrollbar

Every note has a slim minimap rail on its right edge — a fingerprint of the document:

  • Colored dashes mark blocks containing smart tags, in the tag’s own color. Grey dashes are completed blocks; plain structural blocks get faint placeholder dashes so the rail mirrors the whole document.
  • Red dashes and icons flag overdue work: a calendar icon for overdue dates, a clock for overdue reminders, with counts.
  • Accent dashes mark comment threads.
  • Hover the rail and dashes magnify (dock-style) with a tooltip listing the entities at that spot — open ones first, done ones struck through.
  • Click any dash to jump there; the block flashes so you don’t lose it. A white dash shows where you currently are.

Dash colors and what they mean

Each dash on the rail uses the same color as the pill it represents, resolved from the live entity-type registry so a recolored type appears correctly everywhere. The rules, in priority order:

Dash appearanceMeaning
Tag's own color (project, idea, topic, status, or generic hashtag color)A block containing a hashtag of that type, with open work.
Mention's own color (person, team, company, location, product, or generic mention color)A block containing a mention of that type, with open work.
Red dash + clock icon + numberA block with one or more overdue reminders. The number is the overdue count; the clock icon distinguishes reminders from plain dates.
Red dash + number (no clock)A block with an overdue date (a non-reminder date pill that has passed its due instant).
Purple/accent dash + clock iconA block containing an active (running) reminder that is not yet overdue.
Accent-color dash (no entity entities)A comment thread anchored in the document at that position.
Grey dash, reduced opacityA block that is completed — the task is checked, the block has a #complete status pill, or every item in a TodoList/Kanban is done.
White dashYour current scroll position (center of the visible viewport).

Plain future dates (not overdue, not part of a reminder) do not get a dash — only dates that need attention appear on the rail.

Hover magnification and the tooltip

Move your pointer into the rail column and dashes nearest the cursor expand sideways (macOS Dock-style magnification). The focused dash — the one closest to the cursor while the pointer is over the dash column — shows a tooltip to its left listing every entity in that block:

  • Open (pending) entities appear in their own color with their type icon.
  • Completed entities appear greyed, struck through, and with a green check mark.
  • The tooltip shows up to eight entities; additional ones are summarised as "+N more".
  • When a block has both open and done entities, the header line shows the counts: "2 open · 1 done".
  • For comment dashes, the tooltip shows the quoted text and the last reply body.
  • At the bottom of every tooltip: "click to jump" (or "overdue · click to jump" for overdue blocks).

Comments on the minimap

Yes — every open comment thread gets its own accent-color dash on the rail, placed at the vertical center of the highlighted text. Resolved threads appear at reduced opacity. Clicking a comment dash scrolls to and flashes the commented block, then opens the thread popover anchored to the highlighted span.

The command palette

⌘K opens search across everything: notebooks, blocks, hashtags, mentions, files, and app commands. Pressing Enter on a tag opens its lens and flashes it. You can stack multiple tags and toggle between matching all or any of them — instant ad-hoc filtered views.

Search scope, ranking, and scope hints

The command palette searches the entire vault — not just the current notebook. Results come from three pools searched in parallel:

  • Blocks — full-text search across every block in every notebook (up to 25 hits, debounced at 120 ms). Results show the block text and the notebook it lives in.
  • Entities — every @mention, #hashtag, and date pill in the vault. Entity hits are sorted first by type order (Person → Team → Company → Location → Product → Mention → Project → Idea → Topic → Status → Tag → Date), then by how many notebooks reference them. Up to 12 entity hits show per query.
  • Files — attachment names from every notebook, matched by substring (up to 8 hits). Selecting a file hit opens the notebook that contains it.

Matching is substring AND: every space-separated token must appear somewhere in the item's text (case-insensitive). There is no wildcard syntax — partial tokens match because the test is includes, not exact equality.

Two scope hints narrow entity results without affecting block or file search:

  • Type @ at the start of your query to restrict entity results to @mentions only.
  • Type # at the start of your query to restrict entity results to #hashtags only.

When the query is empty the palette shows your six most recent notebooks and the core quick-actions (New notebook, New from template, Open Settings, switch Theme).

Nothing here is manual. There are no refresh buttons in Nomima — lenses, Home, the graph and the briefs all update on the fly as you write, complete and import. The only manual action that exists is the deep “Re-index” in Settings → Vault, and you should rarely need it.
