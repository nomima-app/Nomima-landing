MCP server

Let your other AI tools see your notes. Nomima ships a built-in MCP server so apps like Claude Code, Claude Desktop and Cursor can search, read — and optionally write — your vault.

What is MCP?

The Model Context Protocol is an open standard that lets AI applications connect to data sources. Nomima speaks it natively: enable the server and any MCP-capable tool on your Mac can ask Nomima questions like “search the vault for X” or “list open tasks” — instead of you copy-pasting notes around.

Local only: the server listens on 127.0.0.1 — your own machine. It is never reachable from the network, every request needs a secret token, and write access is a separate opt-in switch.

Finding it

  1. The server is on by default, at http://127.0.0.1:4123/mcp. Open Settings → MCP Server to see its status — the Configuration tab — and toggle it off any time.
  2. Copy the bearer token shown (masked until you reveal it). Use Rotate Token any time to invalidate old credentials instantly.
  3. Optionally toggle Allow writes if you want external tools to be able to create and edit notes — leave it off for read-only access.

Connecting a client

Point any MCP client at the URL with the token as a bearer credential. For Claude Code, for example:

claude mcp add --transport http nomima \
  http://127.0.0.1:4123/mcp \
  --header "Authorization: Bearer YOUR_TOKEN"

Or in a JSON-based client config:

{
  "url": "http://127.0.0.1:4123/mcp",
  "headers": { "Authorization": "Bearer YOUR_TOKEN" }
}

What clients can do

Read tools (always available while the server is on)

ToolWhat it returns
search_notesFull-text search across the vault
list_notebooksAll notes with previews
get_noteA complete note’s content
knowledge_graphEntities and their relations
list_tasksOpen and dated checkboxes, chronologically
list_remindersTracked reminders with due times
get_editor_schemaNomima’s block catalogue, so clients write well-formed notes

Write tools (only when “Allow writes” is on)

ToolWhat it does
create_noteCreates a new notebook from a title and Markdown body
append_to_noteAppends Markdown to an existing note
add_blocksInserts structured Nomima blocks — checklists, callouts, tables, code, reminders and more

Watching the activity log

The Logs tab keeps a live log of every request: which client (Claude, Cursor, …), what it called, whether it succeeded, and how long it took — click any entry for the full request and response. The log window is configurable (50–1,000 entries) and one click clears it. If you ever wonder what an external tool just did to your vault, the answer is right there.

Log entry fields

Each row in the Logs tab represents one complete request/response cycle. Clicking a row expands the full detail. The fields shown are:

FieldWhat it contains
TimestampUnix epoch in milliseconds (timestamp_ms) — shown as a human-readable time in the UI.
ClientThe name the connecting tool reported when it first initialised the session (e.g. claude-code, cursor). Empty if the client has not yet sent an initialize request.
MethodThe JSON-RPC method name — for tool calls this is tools/call; for listing tools it is tools/list; shown as ? for requests that could not be parsed or that were rejected before parsing.
Query previewA short human-readable excerpt: the tool name plus an argument preview, or the URI for resource requests.
StatusOne of four values — see the table below.
DurationRound-trip time in milliseconds from request received to response sent.
Request bodyRaw JSON of the incoming request, truncated to 4,096 characters.
Response previewRaw JSON of the outgoing response, truncated to 512 characters.

Interpreting the status

StatusMeaning
okThe request completed and the response contains a result field — the tool ran successfully.
errorThe server processed the request but the response contains an error field — for example, a tool was called with invalid arguments, or the vault returned no match. Expand the entry to read the error message and code.
unauthorizedThe Authorization header was missing or the bearer token did not match. The server returns HTTP 401; no tool ran. Use Rotate Token and update your client config if you see this unexpectedly.
parse_errorThe request body was not valid JSON-RPC — JSON-RPC error code -32700. The method field is shown as ?. This usually means the client sent a malformed message.

Log retention: the window holds a configurable number of entries (10–1,000, default 200). When the window is full the oldest entry is evicted automatically. Click Clear to wipe all entries immediately. The log is in-memory only — it does not persist across app restarts.

Things people do with it

  • Ask Claude Code to check project notes for context while coding — “what did I decide about the API design? check my Nomima notes”.
  • Have an agent file meeting summaries or research results directly into the vault as proper notes.
  • Query open tasks from any MCP-capable workflow tool.
