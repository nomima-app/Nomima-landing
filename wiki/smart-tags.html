Mentions & hashtags

Smart tags are how Nomima understands who and what your notes are about. Tag once while writing — every view, lens and graph updates automatically.

What smart tags are

A smart tag is a small colored pill embedded in your text. There are two families:

  • @mention — who: people, teams, companies, places, products.
  • #hashtag — what: projects, ideas, topics, statuses.

Every tag becomes an entity in your Knowledge Graph. Tag @Sara in five different notes and Nomima knows they’re all about the same person — her Smart View lens, graph node and entity card collect everything in one place.

Creating a tag

  1. Type @ or # followed by a name. A suggestion menu appears with matching entities you’ve used before.
  2. Press Enter or Tab to accept the highlighted suggestion — Tab always accepts the suggested tag, so you can type two letters and let Nomima finish the name.
  3. If the name is new, choose Add from the menu to create a fresh entity.
  4. Press Esc to dismiss the menu and keep the text plain.

Ghost suggestions: as you type ordinary text, Nomima may show a faint dashed preview of a tag it recognizes (type “Mo” and see @Moe ghosted ahead) with a small ⇥ hint. Press Tab or click it to convert; Esc dismisses it for that spot. You can turn this off with Settings → Smart Tag Types → “Suggest Smart Tags as I type”.

How ghost suggestions work

The faint dashed preview that appears as you type has two sources, checked in this order:

  1. Predictive (instant, mid-word): as soon as you have typed 2 or more characters of a word, Nomima checks your own Knowledge Graph entities and a bundled world gazetteer (~1 000 well-known companies, technologies, products, places and given names). If there is a match, a ghost chip and ⇥ hint appear immediately — no network request, no waiting for you to finish the word. This path produces both @mention and #hashtag ghosts for entities already in your vault, and @mention-only ghosts for the world gazetteer. The world gazetteer path requires the typed prefix to start with an uppercase letter, so common lowercase prefixes don't interrupt ordinary prose.
  2. Backend fallback (whole-word, async): after you finish a word (caret at word end), the backend's entity recognizer also runs across the whole note. If it finds a span ending exactly at the caret that the predictive path didn't catch, it shows a ghost for that too. This covers novel proper nouns and NER hits not yet stored in your vault.

Ghost suggestions cover @mentions and #hashtags only. Date pills are handled by a separate detector that fires when a complete date pattern is typed (no ghost preview — the date converts immediately on completion). Link and email pills convert automatically on Space — they never show a ghost.

Pressing Esc suppresses the ghost at the current caret position only — it reappears if you move the caret or change the word. Permanently dismissing a suggestion (so it never underlines again) is done via the suggestion confirm card: hover the dotted underline and click ✕.

Turn ghosts off entirely in Settings → Smart Tag Types → "Suggest Smart Tags as I type".

Custom types

You’re not limited to the built-ins. In Settings → Smart Tag Types you can:

  • Add a new type to either family — pick a label, an icon from the icon gallery, and a color.
  • Edit built-ins — change a type’s icon or color (its family is fixed).
  • Delete custom types you no longer need — nothing is lost: tags of that type fall back to plain mentions or hashtags, keeping their text. Reserved types like Status can’t be removed.

Custom types get their own color in the Knowledge Graph legend and their own grouping in search — they’re first-class citizens.

Icon gallery for custom types

When you create or edit a type in Settings → Smart Tag Types, you choose from a fixed gallery of 21 icons drawn from the Lucide icon set: user, users, building, map pin, package, briefcase, lightbulb, book, flag, tag, hash, star, heart, folder, target, zap, globe, bookmark, bug, circle-check, cpu. You pick a color using a standard color picker (any hex color). Custom icon uploads are not supported — the gallery is the full set available.

Entity cards & attributes

Click any mention or hashtag pill to open its entity card — the hub for that person, project or thing (date pills open a date picker instead — see Dates & reminders):

  • Appears in — every notebook and block that references the entity, deduplicated and clickable.
  • Attributes — add arbitrary fields to an entity: an email and role for a person, a URL for a company, anything you like. Attributes are stored durably with the entity, not with any single note.
  • Avatars & favicons — upload a photo for a person (compressed automatically) and it appears on that person's pill everywhere in your notes, replacing the default icon. Give a company, product or technology entity a website URL and Nomima fetches the site's favicon automatically (parsing <link rel="icon"> in the page, falling back to /favicon.ico) — the favicon then shows on every pill for that entity too. Both are cached in your vault; no external request is made after the first fetch.
  • Type schema — in Settings → Smart Tag Types you can define which attribute fields a type should have, so every Person card offers the same fields by default.

How "Appears in" works

The Appears in tab of an entity card lists every block in your vault that contains that pill, one row per block. Each row shows the block's text preview and its notebook title; clicking it opens that notebook and scrolls to the block.

The list is deduplicated at the block level: if the same pill appears multiple times in one paragraph block, that block still produces only one row. It is ordered newest-created first. Blocks that are task list titles are excluded from the list (they appear in the lens view instead). A block is shown as struck through when it carries a #complete status pill, a checked task, or a completed reminder.

Editing entity attributes

The entity card's Edit tab shows every attribute field defined for that entity's type. Interaction is click-to-edit:

  • Edit a value — click the field's value (or the pencil hint when empty). An inline input appears; press Enter or click away to save. Press Escape to abandon. URL / email / phone fields use the matching <input type> so your browser's native validation applies.
  • Add a field — click + Add field at the bottom of the Edit tab. A row appears with a label input, a data-type dropdown, and a ✓ confirm button. Press Enter or click ✓ to save; press Escape to cancel. The new field is added to the type schema and therefore appears on every entity of that type, not just this one.
  • Delete a field — click the × that appears on the right of each field row. This removes the field from the type schema and deletes its saved value across all entities of that type. It cannot be undone from the card — re-add the field in Settings if you need it back.

Photo / favicon are not fields. The entity avatar (person photo or company favicon) is set via the header of the card, not via the attribute field list. Click the avatar area to upload a photo (up to 8 MB; auto-compressed before storing). Give a Company, Product or Technology entity a Website field and fill it in — Nomima fetches the site's favicon automatically and shows it on every pill for that entity from that point on. Both are cached in your vault.

Type-level attribute schema in Settings

The fields you add via the entity card belong to the entity's type, not to that single entity — so they appear on every entity of that type. You can manage these schemas directly in Settings → Smart Tag Types without needing to open any entity card:

  1. Open Settings → Smart Tag Types and find the type you want to configure (e.g. Person).
  2. Click the pencil icon on the type row to open its edit form.
  3. Scroll to the Attribute fields section at the bottom of the form. Any fields already defined are listed here with a label input, a data-type dropdown and a trash button each.
  4. Click + Add field to add a new field: give it a name and pick a type, then click Save. The field appears on every entity of this type from now on.
  5. Click the trash button next to a field to remove it from the type's schema — and from every entity's stored data. Use with care.

Built-in types (Person, Company, Project, etc.) come with sensible defaults — Person offers Name, Email and Phone; Company and Product offer a Website field. You can add, rename or remove fields on built-ins just like custom types.

Link & email pills

URLs and email addresses you type also become pills automatically — no @ or # needed.

  • Type a URL (anything starting with https://, http://, www., or a bare domain with a common TLD like .com / .io / .dev / .ai) and press Space — it converts to a link pill.
  • Type an email address (local@domain.tld) and press Space — it converts to an email pill.
  • Pasted URLs and emails also become link/email pills automatically.
  • Clicking a link pill opens the URL in your default browser. Clicking an email pill opens your mail client. Neither shows an entity card.
  • Link and email pills are not tracked in the Knowledge Graph — they don't create entities.
  • Bare hostnames like node.js, version strings like v1.2, or file extensions like README.md are intentionally left as plain text — only real URLs convert.

Which domains auto-convert

Bare domains (no https:// or www. prefix) only convert to link pills when their top-level domain (TLD) is in Nomima's common-gTLD list:

.com  .org  .net  .io  .co  .dev  .app  .ai  .gov  .edu  .info  .xyz  .me  .tv

Anything with an explicit https:// or http:// scheme, or a www. prefix followed by at least one dot, converts regardless of TLD. Strings that look like version numbers (v1.2), file extensions (README.md), or short abbreviations (e.g.) stay plain text even if they contain a dot.

To remove a link or email pill, press Backspace once — all pills are single units and delete in one keystroke. You can also select the pill with arrow keys and press Delete or Backspace.

Pill behavior in the editor

  • Pills are single units: ←/→ step across a pill in one keystroke, and Backspace removes it in one keystroke.
  • Hover a pill for a tooltip showing its type.
  • Pills work everywhere text does — inside task titles, reminder labels, section summaries, callouts and table cells.
  • Pasted text gets tagged too: SmartCopy converts @names, #tags and dates in pasted content into real pills (and is smart enough to leave code comments alone). See Paste, templates & shortcuts.

Why bother tagging?

Tags are the fuel for everything downstream:

  • Smart Views — open the Person lens and see every person with their open work, progress bars, and checkable items; the Entities view is a browsable directory of everything you’ve tagged, with avatars and note counts.
  • Knowledge Graph — entities become glowing nodes connected by what you wrote.
  • Smart Scrollbar — every tag paints a colored dash on the note’s minimap so you can see at a glance what a long note contains.
  • Home — group your daily agenda by @mention or #hashtag instead of by time.
