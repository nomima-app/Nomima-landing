Complete & incomplete

One completion system for everything. Any block — not just checkboxes — can be marked done, from anywhere, and every view in Nomima agrees instantly.

How completion works

A block counts as complete when either of two things is true:

  • It’s a checked task — a checkbox you ticked, or
  • It carries a #complete status tag — a hashtag of the Status type stamped onto the block.

That’s it. There’s no hidden “done” database that can drift out of sync with what you see — completion is always visible right in the note, either as a tick or as a tag.

Two-way sync: these are all the same action. Checking a row in a Smart View edits the actual note; un-checking in the note updates every view. Progress bars, briefs and badges recompute immediately.

Completion in container blocks

  • Task List — shows a live 3/5 progress badge and auto-completes (turns green) when every item is checked. Adding a new item reopens it.
  • Section — stamp #complete on the summary line and the whole section greys out, contents included.
  • Callout — a completed callout swaps its icon for a green checkmark.
  • Kanban — a card completes via its #complete tag or by dragging it to Done; the board completes when every card is done.
  • Gantt — rows complete individually; the chart completes when all rows are done.

Completion cascade — how it spreads

Marking a container complete (or incomplete) automatically propagates down to its children, and checking off the last child automatically marks the container complete. The three cascade layers are:

Task list: parent checkbox drives all descendants

Ticking or unticking a parent task in a Task List immediately sets every indented sub-task (and their sub-tasks, however deeply nested) to match. Un-ticking a single child does not propagate upward — only the parent drives downward.

Stamping #complete on the list title drives the same thing: every task item in that list (including nested sub-tasks) is checked, and the title's #complete pill is removed when you uncheck any task. Removing the tag from the title unchecks every item in the list.

The checkbox and the #complete status tag on a task item are always kept in sync automatically. Ticking the box stamps the tag; typing #complete on the row ticks the box. An existing status tag (like #pending) is replaced rather than left alongside the new one.

Section: completing the summary marks every container child

When a collapsible Section's summary line gains a #complete tag, Nomima cascades completion to every container child — Task Lists, Kanban boards, Gantt charts, and nested Sections — by stamping their titles. Each of those containers then runs its own cascade internally, so a single stamp on a top-level section can tick every task inside every nested list in one go.

Plain paragraphs and bullet lists inside the section are not individually stamped. Instead, the section's completion decoration greys out the entire subtree visually — so prose still reads as done without cluttering each line with a tag.

Removing the #complete tag from the summary reverses this: every container child is reopened by stripping the tag from its title, which in turn unchecks all tasks.

Upstream: finishing all children completes the container

The cascade also works bottom-up. When the last incomplete item in a container (Task List, Kanban, Gantt, or Section) is completed, Nomima automatically stamps #complete on that container's title. When any item is reopened, the tag is removed from the title.

This propagates through the full nesting hierarchy in a single pass: finishing the last task in a nested list can complete the inner list's title, which makes it a fully-complete child of its enclosing Section, which then completes the Section's title too — all at once.

The upstream pass skips containers that have no children, so an empty Task List or empty Section never auto-completes.

What changes when something is done

  • In the note — the text greys out and gets a strikethrough; its date pills turn grey and stop counting as overdue.
  • Smart Scrollbar — the block’s minimap dash turns grey, so a glance at the rail shows how much of a note is finished.
  • Smart Views — the item moves under the collapsed “N completed” toggle; the tag’s progress bar and “open” badge update.
  • Home — it leaves Overdue/Today; when nothing is pending, Home shows your recently completed items instead.
  • Knowledge Graph — the node dims to grey.
  • Tag cards — every lens’s progress bars and open-work badges recount instantly.

Changing your mind

Completion is never destructive. Untick the checkbox or delete the #complete tag and the block returns to open everywhere — overdue dates turn red again, badges recount, and the item reappears in your agendas.
