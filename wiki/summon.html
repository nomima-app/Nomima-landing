Summon

An assistant that actually knows your notes. Summon searches your vault, answers questions, creates notebooks and streams writing straight into your editor — with whatever AI model you chose.

Before you start: Summon needs a Pro license and at least one configured AI model. With a local model (Ollama / LM Studio) it runs fully offline.

Opening Summon

  • Click the glowing button in the bottom-right corner — it breathes quietly when idle and animates while Summon is working, with a badge counting in-flight tasks.
  • Or press ⌘⇧Space anywhere, or type /summon in any note — your current block and selection travel along as context.
  • Summon opens as a compact dock on the right edge; the expand button promotes it to a full-screen view. Close with Esc — the conversation survives, even as you switch notebooks.

From chat into your notes

Summon's answers don't have to stay in the chat:

  • Pin an answer — any reply can be pinned into a notebook as a Section, tagged so you can find everything Summon wrote.
  • Accept proposals inline — when Summon drafts a notebook or template, it shows up as a proposal card with Accept / Reject right in the conversation. Nothing lands in your vault until you accept.
  • Real blocks, real tags — what it writes are genuine Nomima blocks (checklists, callouts, tables, kanban) with working @mentions, #hashtags and dates; its answers even link back to the notes they cite as clickable chips.

Summon isn’t a generic chatbot with your note pasted in — it has tools to work with your vault directly:

Accepting or rejecting block proposals

When Summon drafts a new notebook or template it doesn't write it immediately. It presents a proposal card in the conversation showing the notebook title and a summary of what it will contain. Two buttons appear at the bottom of the card:

  • Accept — creates the notebook, writes the proposed blocks into it, and shows an Open button so you can jump straight to it.
  • Reject — marks the proposal as rejected (shown as dimmed text, no further action taken) and nothing is added to your vault.

Rejecting is permanent for that proposal — it stays in the chat history as a record, but the card is no longer actionable. You can follow up in the conversation to ask Summon to try a different approach.

Agent permissions

Summon’s reach beyond your vault is off by default and controlled in Settings → Summon Agent:

  • Web search — let Summon look things up online (via DuckDuckGo).
  • Filesystem access — let it read and write files in Documents, Desktop and Downloads. Reads run freely; every file write (and shell command) pauses for an inline Allow / Deny right in the chat.
  • Shell commands — strictly allowlist-based: you add the exact commands it may run, and chaining/piping is blocked.

Privacy reminder: when Summon answers a question about your notes, the relevant note content is sent to the AI provider you configured — that’s how the model reads it. Use a local model if you want even that to stay on-device. Nothing is ever sent anywhere when you’re not actively using Summon.

Hard security limits

The following caps are enforced in the Rust backend and cannot be changed from the frontend or by the AI model:

LimitValue
Web page fetch (characters returned to the model)6 000 characters — content beyond this is truncated
Filesystem file read256 KB — content beyond this is truncated
Shell command output (stdout + stderr each)64 KB — output beyond this is truncated
Shell command timeout30 seconds — the command is killed and an error is returned

Filesystem access is confined to your Documents, Desktop and Downloads folders. Every path is canonicalised before the range check, so .. traversal and symlink escapes are blocked. Shell commands run as a bare command name against a fixed allowlist (default: git ls cat grep rg find echo pwd wc head tail npm node python python3 cargo); paths, spaces and shell metacharacters in the command name are rejected outright — there is no sh -c wrapper. Web fetches are restricted to http:// and https:// URLs only.

You can extend the shell allowlist in Settings → Summon Agent. Filesystem and shell access are off by default; web access is on by default.
