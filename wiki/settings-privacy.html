Vault stats & metadata

Settings → Vault shows a live snapshot of your vault's contents. The numbers come directly from the database — no estimation:

StatWhat it counts
NotebooksAll non-deleted notebooks in the vault.
FilesEvery attachment you have imported (count and total size on disk).
Knowledge Graph nodesExtracted entity nodes in the graph index (kg_entity rows).
Vault sizeTotal bytes used by the vault folder — excluding the bundled AI model weights in the models/ subfolder, which are a fixed app asset rather than your data.

The same panel also shows the vault location (the full path on disk) and a Move vault action that copies everything to a new parent folder you pick. After a successful move, Nomima restarts pointing at the new location. The old folder is not deleted automatically — remove it yourself once you have confirmed the move.

Re-index rebuilds the search index and Knowledge Graph from scratch. Use it if search results look stale or the graph is missing entries after an import. It does not delete or move any notes or files.

File text extraction

When you import a file (PDF, Word document, etc.) Nomima extracts its text immediately at import time and stores it alongside the attachment. If extraction failed — for example because a PDF was scanned-only or password-protected — the file's entry in the Files Smart View will show an "unextracted" state. There is currently no per-file re-extraction button; running Re-index (Settings → Vault) will retry extraction for any file whose text is missing as part of the full re-index pass.

Privacy: what stays, what leaves

Nomima is local-first in the literal sense. Your vault — every note, file, comment and graph relationship — lives in a folder on your Mac (under ~/Library/Application Support), stored in an ordinary SQLite database. (Vaults from very early versions that lived in Documents are migrated there automatically.) No account exists, no telemetry is collected, nothing syncs anywhere.

The complete list of network calls

CallWhenWhat’s sent
Update checkOn launch / when you click “Check for updates”A request for a version file — nothing about you or your notes
AI provider callsOnly when you use Summon or the /generate command, with a provider you configuredThe conversation and relevant note context, to your chosen provider (or to a local model: nothing leaves at all)
Summon web searchOnly if you enabled it in SettingsThe search query
PublishingOnly when you publish, update or unpublish a noteThe note’s rendered page (and files you explicitly ticked) to the read.nomima.io service
RSS feedsOnly for feeds you subscribed in an RSS blockA fetch of that feed and its article images (cached locally)
Link previews & faviconsWhen you paste a URL into a Bookmark block or set an entity’s websiteA direct fetch of that page’s metadata or icon

That’s the whole list — and every entry is something you invoked. Offline, Nomima makes zero network requests that involve your data, and everything outside AI, RSS, link previews and publishing works exactly the same.

Summon Agent: security boundaries

The three Summon Agent capabilities — web, filesystem and shell — are each off by default and must be enabled individually in Settings → Summon Agent. Every toggle is enforced in Rust; disabling one from Settings cannot be bypassed by a running conversation. See Summon for the full usage guide.

Hard limits (cannot be overridden from a prompt)

CapabilityLimitDetail
Web fetch6 KB of text per pageHTML is stripped to plain text before the cap is applied. Only http:// and https:// URLs are accepted.
Filesystem read256 KB per fileLarger files are truncated with a …(truncated) marker. The agent can only read and write inside your Documents, Desktop, and Downloads folders — every path is canonicalised before the range check, so .. traversal and symlink escapes are both blocked.
Shell output64 KB (stdout + stderr combined)Output beyond the cap is truncated. Commands run with a 30-second timeout and kill the process if it does not finish in time.
Shell command injectionAllowlist only, no shell interpreterThe agent can only run bare command names (e.g. git, grep) that appear on your allowlist. Paths, spaces and shell metacharacters in the command name are rejected outright — the command is passed as an argv array, never through sh -c.

The default shell allowlist is: git ls cat grep rg find echo pwd wc head tail npm node python python3 cargo. You can add or remove entries in Settings → Summon Agent. Filesystem writes and all shell commands additionally require an interactive Allow / Deny prompt in the Summon panel before they execute — there is no way for the agent to skip this step.

All three capabilities default to off — enable only what you need.

Completely free

Nomima is free — there is no Pro tier, no license to activate, and no upgrade prompts. Every feature is unlocked for everyone:

  • The full editor and every block, Smart Tags, dates & reminders
  • Smart Views, Home, the 3D Knowledge Graph & search
  • Comments, files, PDF export, and all templates — including creating your own
  • The Summon assistant and the /generate command (bring your own AI key, or run a local model)
  • Configuring AI models, the MCP server, and publishing notebooks to the web

No license, nothing to buy. Your data stays on your device; the only things that ever leave your Mac are the notebooks you explicitly publish and any AI requests you make with your own key.

Vault care

  • Back up your vault folder like any other documents — it’s self-contained and portable.
  • Re-index (Settings → Vault) rebuilds search and the Knowledge Graph from your notes if anything ever looks stale.
  • Storage stats on the same panel show exactly where the space goes.
