Writing

Blocks

Type / anywhere in a note to open the block menu, then keep typing to filter it. Press Enter to insert the highlighted block, Esc to dismiss.

Tip: blocks that have a title (Task List, Section, Kanban, Gantt, Dashboard, Widget) drop the caret straight into the title when created — name it, press Enter, and keep writing. Mentions, hashtags and dates work inside those titles too.

Block anatomy

Most rich blocks share a common header row. Understanding it once means you know how every board, chart, diagram, and drawing works.

Collapse chevron

Every rich block (Kanban, Gantt, Mermaid, Math, Drawing, Dashboard, Widget, Bookmark, RSS, Task List, Section) shows a small icon-toggle on the left of its header. At rest the icon identifies the block type (a ChartGantt icon for timelines, a GitBranch icon for diagrams, and so on). Hover over it and it becomes a collapse chevron. Click to fold the block body down to its header row; click again to expand. The content is never destroyed — it stays indexed by the Knowledge Graph even while folded. This collapse is distinct from the Toggle List block, which is a plain inline disclosure without any card chrome.

Note: the collapse chevron is also the block's drag handle. Grab it and drag to reorder the block within the note.

Block caption (title row)

Kanban, Gantt, Mermaid, Math, Drawing, Dashboard, Widget, Bookmark, and RSS blocks share a unified editable caption row at the top. It is a real text field — not a plain attribute — so:

  • @mentions, #hashtags, and dates work inside every caption for free.
  • Typing #complete in the caption marks the whole block done: it acquires an emerald border and a struck title, and counts as complete in Smart Views and the Daily Brief — just like checking off a task.
  • Pressing Enter in the caption moves the caret into the block's content rather than splitting the caption (a caption holds only inline content).

Tip: completing a block via its caption is the fastest way to archive a finished Kanban board, Gantt timeline, or diagram without deleting it.

Task List

A titled checklist card. Create it from the / menu or by typing [] + space at the start of a line.

  • Editable title — name the list; pressing Enter in the title jumps to the first task. @mentions, #hashtags and dates work in the title.
  • Progress badge — the card header shows 3/5 style progress as you check items off.
  • Collapsible — the chevron folds the list down to its title row.
  • Auto-complete — when every task is checked, the whole list turns green and counts as complete. Add a new task and it flips back automatically.
  • Backspace on an empty list deletes the whole list cleanly; arrow keys escape out of it in every direction.

Note: the / block menu is intentionally disabled inside checklist items — checklists stay checklists. Press Enter to leave the list first.

Task List — subtasks and nesting

Press Tab on any task row to indent it under the row above, creating a subtask. Subtasks can be nested at any depth. Shift Tab outdents one level. The header badge shows two counts when subtasks exist: 2/5 for top-level tasks and a second badge (with a corner-arrow icon) for subtasks, e.g. ↵ 3/4. The main progress bar reflects top-level completion only.

The / block menu is disabled inside task rows at every nesting depth — checklists stay checklists. Press Tab past the last non-indentable row to exit the list to the next block.

Ratios are implemented as CSS fr tracks (minmax(0, Nfr)), so columns shrink gracefully on narrow notes and never overflow.

Callout — completion icon

When any block inside a callout carries #complete, the callout's icon badge changes to a green CircleCheck regardless of the callout's type — a clear "done" stamp on the whole box. The type is unchanged internally (the picker still shows it), but the accent color that drives the border, rail, background tint, and icon all flip to emerald until the pill is removed.

Diagram (Mermaid) — editing workflow

A freshly inserted diagram block opens straight into Edit mode: a full-width textarea where you write Mermaid source. The toolbar shows Preview and Copy buttons. Click Preview (or blur the textarea) to render the diagram; click Edit to return to the source. If the source has a parse error, the block shows the error message with an Edit source button — fix the syntax and preview again. Copying a diagram block as text yields its raw source.

Note: Mermaid supports flowcharts, sequence diagrams, ER diagrams, class diagrams, mind maps, pie charts, and Gantt syntax. Check the Mermaid docs for the full language reference.

Equation (LaTeX / KaTeX) — editing workflow

An empty equation block opens in edit mode automatically. Type LaTeX source in the textarea (resize by adding lines); the rendered equation appears as soon as you click the checkmark or press Esc / ⌘ Enter. To edit an existing equation click the rendered output — it flips back to the textarea. Editing the LaTeX while the block carries #complete automatically strips the completion pill in the same undo step, so revising an equation un-completes it.

Drawing (Excalidraw) — resize

The drawing canvas starts at 440 px tall. Drag the resize edge at the bottom of the canvas to set a custom height (minimum 220 px, maximum 900 px). The height is saved in the block and restored on next load. The Excalidraw toolbar is hidden when the block is collapsed.

Boards & timelines

Kanban Board

Draggable cards in three fixed columns — To Do, In Progress and Done. Tagging a card #complete sends it to Done; dragging a card into or out of Done adds or removes that tag for you, so the board and your notes can never disagree. Any other status tag is free-form and simply stays on the card. The header shows an active-of-total count.

Gantt Chart

A timeline of tasks with start and end dates. Switch between Fit (default), Day, Week and Month zoom. Edit mode exposes name, start, end and assignee fields per row; preview mode shows just the chart. Task names take Smart Tags — typing #complete in a task’s name marks it done.

Dashboard & Widget

Dashboard is a live grid of widgets showing stats and trends about your vault; Widget is a single resizable chart or metric you can drop anywhere in a note.

Kanban — status tags and the Done column

The board's one and only automation is the #complete pill. Every other status tag you type on a card (#in-review, #blocked, and so on) is a freeform label that stays on the card and never triggers a column move. Only #complete relocates the card.

  • Adding #complete to a card in any column moves it to Done immediately and remembers which column it came from.
  • Removing #complete from a card in Done flies it back to the column it came from (defaulting to the first column).
  • Dragging a card into Done stamps #complete on it; dragging it out strips the pill — so the board and your tags are always in sync.
  • Adding #complete to the board's title stamps it on every card at once, sending them all to Done. Removing it from the title sends every card back.

Gantt — zoom levels and assignees

The chart rests at Fit view — the whole timeline compressed into one screenful with no horizontal scrolling. The toolbar offers three manual zoom levels:

LevelScaleTick step
Fit (default)Auto — spans the full range6 evenly-spaced labels
Day26 px / dayEvery 7 days
Week10 px / dayEvery 14 days
Month3 px / dayEvery 30 days

Zoom is transient: the chart auto-reverts to Fit a moment after your pointer leaves it. Switch to Edit mode (pencil icon in the toolbar) to see and edit each task's name, start date, end date, and assignees. Assignees are free text (e.g. Dana, Sam). The bar itself is still draggable in Edit mode — drag the bar to move the span, drag either edge to resize it, or click an edge to open a date picker.

Typing #complete in a task name strikes and greys the task. Adding #complete to the Gantt's title marks every task done at once; removing it reopens them all.

Press Enter on a task with text to add a new task below it. Press Tab from any task to exit the chart to the next block.

Dashboard and Widget — resize

Dashboard starts as a 4-column, 1-row bento grid (1 to 6 rows). Drag the bottom edge of the grid to add or remove rows (snaps to whole square rows; can never shrink below occupied content). The grid is full-width only — insert it at the top level, not inside a column or callout.

Widget is a single chart or metric that can sit anywhere (including inside a Columns block). Drag the corner handle to resize it freely (minimum 180 × 120 px, maximum height 640 px). The widget shrinks to fit a narrowing parent column without overflowing.

Both blocks share the unified caption row: the title accepts smart tags, and typing #complete marks the whole block done.

Bookmark / Video — offline image caching

When you paste a URL into a Bookmark block, the Rust backend fetches the page's OpenGraph metadata (title, description, site name, cover image, favicon) and copies the images into your vault. The card renders those vault copies via the asset: protocol, so it looks right offline and the webview never touches the network again. Clicking the card opens the URL in your OS browser. The block stores everything needed to render — url, title, description, siteName, imagePath, faviconPath — in the node, so the preview survives note export and re-import.

For video URLs, Vimeo embeds a playable inline player. YouTube shows a thumbnail with a play button that opens the video in your browser — YouTube's player requires a valid HTTP referrer that a packaged offline app cannot supply, so in-app playback is not available for YouTube.

RSS Feed — topics, custom URLs, and pinning

The RSS block has two feed sources that can be combined:

  • Topic catalog — a curated grid of hashtag topics (e.g. #tech, #science). Click up to 3 topics from the picker grid. Articles are fetched privately and cached on your device; browsing them adds nothing to your Knowledge Graph.
  • Custom URLs — paste any RSS/Atom feed URL into the text field and press Enter or the + button to add it. Multiple URLs are listed below the field and can be removed individually.

Once configured, click Fetch articles to load the feed. While articles are showing, a Settings button (gear icon) reopens the picker to change your selection.

Pinning: while reading an article inside the block, click Pin to save it as an RSS Memory block inserted immediately after the feed block. Only pinned memories are ingested into your Knowledge Graph and become taggable, searchable, annotatable entries — the rest of the feed is ephemeral. An alternative is Add as block, which inserts the article as a Section block (headline as title, body as content) that you can then edit freely.

Offline: the block shows cached articles when the device is offline with a banner saying "Offline — showing saved articles". A failed refresh never blanks the article list.

Block Reference — creation and preview

Type / and choose Block Reference, or use the [[ shortcut where supported, to open a reference picker overlay. The picker has two levels:

  • Top level — search across all notebooks. Select a notebook to insert a notebook-level reference, or drill into it to pick a specific block.
  • Block level — after drilling into a notebook, search within it and pick the exact block.

The reference appears inline as a pill with the target's label and a link icon. Hover the pill to see a hover card with the notebook title and a short preview. Click the pill to navigate directly to that notebook (or scroll to the specific block).

File — size limit, preview, and Knowledge Graph

Files can be attached via the / menu, by dragging a file onto a note, or via the Files Smart View. Each file is limited to 10 MB per file. The file is copied into your vault and text is extracted automatically (PDFs, Office documents, plain text). The extracted text is embedded and joins your Knowledge Graph and full-text search — the file's content is searchable even when the file itself is closed.

Click a file pill to preview it inline (images show directly; PDFs open in a sandboxed reader); click the external-link icon to open the file in its default macOS app. Deleting a file from the Files Smart View purges the vault copy, the extracted text, the embeddings, and removes the pill from any open editor in one operation.

Table — navigation and exit

Tab moves to the next cell (right, then down to the next row); Shift Tab moves to the previous cell. Arrow keys move between lines inside a multi-line cell. To exit the table entirely: press ↓ from the last row's last cell, or ↑ from the first row — the caret steps out to a paragraph above or below the table (creating one if the table sits at the document boundary).

ASMR Block — mute toggle and sound settings

Each ASMR block has a per-block mute toggle (speaker icon) in its header. Click it to silence that block without affecting other blocks or the global sound setting. The icon shows a crossed-out speaker when muted. The sound pack (the set of key variants and timing) is configured globally in Settings → Sound.

Typing Enter on a non-empty line adds a new line within the block; pressing Enter on an empty trailing line exits the block to a plain paragraph in the main editor. Arrow keys navigate in and out of the block past its non-editable header.
